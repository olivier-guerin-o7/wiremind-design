What would a more considered Market View look like? This prototype is my answer — incomplete by design, but grounded in real observations about how analysts navigate dense data under time pressure.

Non-expert perspective This redesign was approached without deep domain expertise in revenue management. I invested enough time in the widget to form meaningful design hypotheses — but almost certainly missed important context or misread some mechanics. Each proposal should be validated against real operational knowledge before acting on it.
Sizes are off This prototype runs at a magnified scale — larger than production dimensions. Small content (figures, labels, dense data) would likely break visually when reduced to real size. This was a deliberate tradeoff: working at this scale let us test and iterate on design decisions faster within a limited timeframe. A full rework at real dimensions would be needed to validate these proposals in context.
Charts and radar: macro pass only I deliberately scoped out deep reworks of the loading curves and radar zones. Given the time available, I prioritised macro layout and interaction decisions over fine-grained data visualisation refinements — which would require closer domain and data expertise to do well.
proposal
Color system audit AU Grid encodes price tier via colour. Competition encodes threat level. If both use red, the analyst will misread across panels. Explicit visual separation required — by hue family, shape, or labelling.
proposal
Max Leg as a standalone section Contextual metadata. Useful once to orient, then forgotten. Move to macro bar or tooltip — not a panel-level section.
proposal
Optimization Run timing detail Informational, not decisional. The analyst doesn't need the exact booking threshold trigger to decide. Replace with a compact status badge.
proposal
Per-panel section headers Labels that name what's already visually obvious. Consume vertical space without contributing to the decision loop.
proposal
Macro bar — persistent context layer A fixed strip always visible at the top: current D-X, load factor %, active bucket, active alerts, key demand model values. Eliminates hunting for basic orientation data at the start of every market check.
proposal
Forecast↔Optim gap indicator The gap between where you're going (Forecast) and where you could go (Optim) is the core decision. Surface it as an explicit number — not just two lines on a graph. This gap is the revenue opportunity on the table. It should be impossible to miss.
proposal
Max Leg → macro bar Out of its own section, into the persistent header where it serves as always-visible orientation without consuming panel space.
proposal
Demand Matrix key values → macro bar Currently buried in a sub-tab. As the foundation both Forecast and Optim are built on, key values should be scannable at all times. If the Demand Matrix is wrong, every line in Loading Curves is wrong.
proposal
Optimization Run timing → status badge in header Still present, not lost, but no longer competing for panel space.
proposal
AU Grid larger, Loading Curves + Competition smaller AU Grid is the most information-dense and highest-stakes panel. With computed indicators from the other panels added, it earns more horizontal space. The others become supporting context — always visible, compact, full-size on demand.

proposal
Ancillaries section Add-on pricing (bags, meals). Specialised use case, not part of the yield decision loop for most sessions. Move behind a dedicated tab.
proposal
Computed threat indicators per bucket row Per bucket row: badge showing "3 red competitors", cheapest competitor delta vs. current bucket price. Analyst doesn't need to cross-reference Competition for a routine threat check — one decision, one place to look.
proposal
Lightweight friction for bucket commits A brief confirmation step or short undo window before an AU change is live. Currently no structural protection against a mis-entry on a no-undo surface where changes are visible to customers within minutes.
proposal
Forecast & Optim lines visually dominant Currently dotted/grey — conventionally "uncertain". But they are the most actionable signals. They should look like it: thicker, stronger colour, foregrounded. The most actionable lines are the quietest. Fix the inversion.
proposal
Reduce default overlay set 10+ toggles always visible creates decision overhead before reading the chart. Default: Current + Reference + Forecast + Optim. Everything else off by default, accessible via toggle.
proposal
Split into two explicit modes: History vs. Forecast History mode: Current + Reference (what happened). Forecast mode: Forecast + Optim (where it's going). Separates past fact from future projection — reduces visual competition and cognitive load per read.
proposal
Most overlay toggles from default view Always-visible toggles create decision overhead before the analyst even reads the chart. Move non-default overlays behind "more" — reduce the default visual menu to what's needed for 95% of sessions.
proposal
Radar annotated in situ Keep the radar's spatial dimension (time proximity is irreplaceable). Overlay computed indicators directly: threat count badges by level, cheapest competitor delta vs. your price, carrier flags. Radar becomes annotated rather than raw — talkative without being a table.
proposal
Computed indicators (automated threat rules) Any rule readable deterministically from the radar can be computed: threat count by colour, cheapest competitor, carrier flags, "more aggressive this week" trend. Automate what is 100% consistent — human judgment reserved for ambiguous cases.
proposal
Competitor data as collapsible table (drawer) The radar gives spatial context; the fares history gives temporal context. A structured table of competitor fares — available on demand via a drawer — gives precision for the analyst who needs exact values, not just visual position.
